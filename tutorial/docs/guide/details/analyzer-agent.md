# Analyzer Agent 详细指南

`AnalyzerAgent` 负责把评测现象转成诊断结论。

## 核心职责

- 读取评测结果
- 归纳失败模式
- 生成分析报告
- 提出后续优化建议

## 进入它之前通常要准备什么

Analyzer 往往依赖：

- 已完成的评测结果
- 分析模型服务配置
- 相应的结果路径或报告输出路径

## 它的输入和输出可以怎么理解

输入通常是：

- Judger 的评测结果
- 样例输出
- 分析模型配置

输出通常是：

- 问题模式总结
- 分析报告
- 可供 Obtainer 或 WebCrawler 继续使用的优化方向

## 在闭环中的位置

Analyzer 位于“评测之后、数据动作之前”。它的作用是回答：问题到底出在哪里。

## 使用时最该关注什么

- 评测结果路径是否正确
- 分析模型是否足够强
- 输出的报告是否具有可执行性
